📦 OpenClaw 隨身碟可行性報告

評估日期：2026-03-30 | 報告人：蝦管

🔍 現狀評估

項目 現況 說明
核心程式 npm 全域 需 Node.js 22+ 運行
資料大小 ~470 MB 含 agents、workspace、credentials
設定位置 ~/.openclaw/ 可透過 OPENCLAW_HOME 自訂
攜帶性 ⚠️ 路徑依賴 新電腦需設定環境變數

🗂️ 三種可行方案

方案 描述 難度 限制
A. 全資料攜帶 隨身碟放 ~/.openclaw ⭐ 最簡 需新電腦已裝 Node
B. Docker 容器 隨身碟放 Portable Docker ⭐⭐⭐ 需 USB 3.0+
C. 完全自含 從原始碼編譯 ⭐⭐⭐⭐ 複雜，磁碟大

⚡ 方案 A 詳解（推薦）

📁 建議資料夾結構

隨身碟/ └── .openclaw/ ├── agents/ # GSwitch 團隊設定 ├── workspace/ # 任務資料 ├── credentials/ # API 金鑰 ├── config.json # 設定檔 └── memory/ # 記憶資料

🚀 新電腦設定流程

步驟 1：確認 Node.js 版本
node -v # 需 v22.14+ 或 v24+
步驟 2：設定環境變數
# Windows (PowerShell) $env:OPENCLAW_HOME = "D:\USB\.openclaw" # Linux/macOS export OPENCLAW_HOME="/mnt/usb/.openclaw"
步驟 3：啟動 OpenClaw
openclaw start

⚠️ 方案 A 限制說明

限制項目 說明 解決方案
Node.js 目標電腦需已安裝 攜帶 Node 安裝檔或用Portable版本
效能 USB 2.0 會慢 3-5x 使用 USB 3.0+ 或內接 SSD
系統相容 Linux/macOS 可直接用 Windows 需 WSL2
安全性 金鑰在隨身碟 使用後拔出，妥善保管

🎯 結論

方案 A（全資料攜帶）是最快、最簡單的方案。

建議少爺先用隨身碟設定 OPENCLAW_HOME，測試在不同電腦的移轉效果。

若未來有更高度隱私需求，再評估方案 C（完全自含編譯）。
